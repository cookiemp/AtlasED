Summary

The useState hook is React's primary mechanism for adding state to functional components. It returns a state value and a setter function, enabling components to remember values between renders. Understanding the nuances of state initialization and update patterns is crucial for avoiding common bugs.

Key Concepts

  • 1 State persists across re-renders but initializing with a function prevents unnecessary calculations
  • 2 State updates are batched for performance - multiple setState calls may trigger a single re-render
  • 3 The setter function accepts a callback when the new state depends on the previous state

Code Example 

const [count, setCount] = useState(() => 0);

const increment = () => {
  setCount(prev => prev + 1);
};

Remember

Always use the functional update form when the new state depends on the previous state to avoid stale closure bugs.
