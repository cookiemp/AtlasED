# 🌍 Project Title: **Atlased**

### **The Core Concept**
**Atlased** is a desktop application that acts as a "learning wrapper" for YouTube.
*   **The Problem:** YouTube is designed for *engagement* (keeping you watching).
*   **The Solution:** Atlased is designed for *retention* (helping you learn).
*   **How:** It takes a YouTube playlist, strips away the distractions, and uses Artificial Intelligence to turn the video content into a structured, interactive course with notes, quizzes, and connections.

---

# 📖 The "Atlas" Dictionary (Terminology)
To make the app feel cohesive, we use specific terms in the code and the UI.

| Standard Term | **Atlased Term** | Meaning |
| :--- | :--- | :--- |
| **Playlist/Course** | **Expedition** | A collection of videos on a specific subject (e.g., "Python for Beginners"). |
| **Video** | **Waypoint** | A single stop on the journey (e.g., "Video #4: Variables"). |
| **Summary/Notes** | **Field Guide** | The text, code snippets, and key points generated by AI for that video. |
| **Knowledge Graph** | **The Atlas** | The visual map showing how topics connect across different videos. |
| **User Progress** | **Traversing** | The act of completing videos and quizzes. |
| **Spaced Repetition** | **Memory Checkpoint**| Scheduled reviews of past Waypoints to ensure long-term retention. |

# 👤 The User Journey (How it works)

**Step 1: The Launch**
The user opens the app (desktop). It looks like a sleek, dark-mode dashboard. They see their active Expeditions and any "Memory Checkpoints" due for the day.

**Step 2: The Input**
They paste a YouTube Playlist link. 
*   **System Action:** The app fetches titles, thumbnails, and durations, creating a new "Expedition" in the local database.

**Step 3: The Journey (Watching)**
The user clicks a Waypoint. The player is focused, with the "Field Guide" and "Atlas" easily accessible.
*   **System Action:** App fetches the transcript (using `youtube-transcript`) and generates initial AI notes/quizzes if not already done.

**Step 4: Active Engagement**
*   **Mid-stream Quizzes:** Video pauses at AI-determined "natural breaks" for quick comprehension checks.
*   **Field Guide:** Dynamic Markdown notes appearing below the player.

**Step 5: The Connection**
Finishing a video updates "The Atlas." New connections are drawn between topics (e.g., "Variables" linked to "Data Types").

**Step 6: The Review**
The app notifies the user when a Waypoint is ready for a "Memory Checkpoint" (Day 1, 3, 7, 14), reinforcing the knowledge.

# 🏗️ Technical Architecture (The Stack)

This is a **Local-First** app using the **BYOK (Bring Your Own Key)** model for Gemini.

### **1. The Shell & UI**
*   **Electron + React + Vite:** For a high-performance desktop experience with fast development cycles.
*   **Vanilla CSS:** Custom luxury dark-mode aesthetic (Deep blues, gold accents, glassmorphism).

### **2. The Brain (Google Gemini API)**
*   **Long Context:** Leveraged for analyzing entire playlist transcripts to find cross-video connections.
*   **BYOK:** Users provide their own API key (Free tier supported).

### **3. The Memory (SQLite)**
*   **`better-sqlite3`:** For fast, synchronous database operations.
*   **Drizzle ORM (Optional):** For type-safe schema management.

### **4. Extraction**
*   **`youtube-transcript`:** Lightweight and focused for fetching video captions.
*   **YouTube IFrame Player API:** For streaming video without local storage.

# ⚙️ Detailed Feature Specifications

### **Feature A: The Ingestion Engine**
1. User inputs URL.
2. App gets video IDs/titles via YouTube Data API.
3. App fetches transcript via `youtube-transcript`.
4. Data saved to `Expeditions` and `Waypoints` tables.

### **Feature B: AI Field Guides**
1. Prompt Gemini with the transcript.
2. Request: Executive Summary, 5 Key Concepts (with tags), and Code Examples.
3. Render as beautiful Markdown.

### **Feature C: Mid-Stream Quizzes**
1. AI identifies 2-3 logical timestamps for questions.
2. Video pauses automatically at these timestamps.
3. User must answer to proceed (Active Recall).

### **Feature D: The Atlas (Knowledge Graph)**
1. AI extracts 3-5 tags per video (e.g., "Recursion", "Big O").
2. Graph visualizes connections between Waypoints based on shared tags.
3. Library: `react-force-graph`.

### **Feature E: Memory Checkpoints (SRS)**
1. Tracks quiz performance in `QuizAttempts`.
2. Schedules reviews based on a simplified Spaced Repetition algorithm (1, 3, 7, 14 days).
3. "Review Due" indicators on the dashboard.

# 📝 Database Structure (Schema)

**1. Table: `Expeditions`**
*   `id`: UUID
*   `title`: Text
*   `thumbnail_url`: Text
*   `created_at`: Timestamp

**2. Table: `Waypoints`**
*   `id`: UUID
*   `expedition_id`: FK
*   `youtube_id`: Text
*   `title`: Text
*   `thumbnail_url`: Text
*   `duration_seconds`: Integer
*   `order_index`: Integer (position in playlist)
*   `last_watched_pos`: Integer (for resume)
*   `transcript_text`: Text
*   `is_charted`: Boolean
*   `next_review_at`: Timestamp

**3. Table: `FieldGuides`**
*   `waypoint_id`: FK
*   `markdown_content`: Text
*   `quiz_data_json`: JSON (Questions + Timestamps)

**4. Table: `Tags` & `WaypointTags`**
*   `id`: UUID
*   `name`: Text (Unique)
*   Mapping table to link Tags to Waypoints for the Atlas graph.

**5. Table: `QuizAttempts`**
*   `id`: UUID
*   `waypoint_id`: FK
*   `is_correct`: Boolean
*   `attempted_at`: Timestamp

**6. Table: `Settings`**
*   `key`: Text (Primary Key)
*   `value`: Text (Encrypted for sensitive data)
*   `updated_at`: Timestamp

# 🤖 AI Prompt Templates

### **Prompt A: Field Guide Generation**
```
You are an expert educational content creator. Analyze the following video transcript and create a comprehensive Field Guide.

**Transcript:**
{transcript_text}

**Output Format (JSON):**
{
  "executive_summary": "2-3 sentence overview of the video content",
  "key_concepts": [
    {
      "title": "Concept Name",
      "explanation": "Clear explanation in 2-3 sentences",
      "tags": ["tag1", "tag2"]
    }
  ],
  "code_examples": [
    {
      "language": "python",
      "code": "example code here",
      "explanation": "What this code demonstrates"
    }
  ],
  "key_takeaways": ["Takeaway 1", "Takeaway 2", "Takeaway 3"]
}

Focus on clarity and actionable learning. Extract 3-5 key concepts with relevant tags for knowledge graph connections.
```

### **Prompt B: Quiz Generation**
```
You are an expert educational assessor. Based on the following video transcript, create comprehension quizzes that test understanding at natural break points.

**Transcript:**
{transcript_text}

**Output Format (JSON):**
{
  "quizzes": [
    {
      "timestamp_seconds": 180,
      "question": "What is the purpose of X?",
      "options": ["Option A", "Option B", "Option C", "Option D"],
      "correct_index": 0,
      "explanation": "Brief explanation of why this is correct"
    }
  ]
}

Rules:
- Create 2-3 quizzes per video
- Place quizzes at logical topic transitions (not mid-sentence)
- Questions should test comprehension, not memorization
- All options should be plausible
```

### **Prompt C: Cross-Video Analysis (Atlas)**
```
Analyze these video transcripts from a learning playlist and identify conceptual connections.

**Videos:**
{array_of_transcripts_with_titles}

**Output Format (JSON):**
{
  "connections": [
    {
      "source_video_id": "video_1_id",
      "target_video_id": "video_3_id",
      "shared_concepts": ["Recursion", "Call Stack"],
      "relationship": "Video 3 builds upon recursion concepts introduced in Video 1"
    }
  ],
  "learning_path": ["video_1_id", "video_2_id", "video_3_id"],
  "prerequisite_map": {
    "video_3_id": ["video_1_id"]
  }
}
```

# 🛡️ Error Handling & Fallbacks

### **Scenario 1: Transcript Unavailable**
*   **Cause:** Video has no captions, captions disabled, or private video.
*   **Fallback:** 
    1. Show warning: "No transcript available for this Waypoint."
    2. Allow manual notes entry.
    3. Skip AI-generated Field Guide and Quizzes.
    4. Mark `transcript_text` as `NULL` in database.

### **Scenario 2: Gemini API Rate Limit**
*   **Cause:** Free tier exceeded (15 RPM, 1M TPM).
*   **Fallback:**
    1. Queue failed requests with exponential backoff.
    2. Show toast: "AI generation paused. Retrying in X seconds."
    3. Process queued items on next successful request.
    4. Store pending tasks in `PendingAITasks` table.

### **Scenario 3: YouTube API Quota Exceeded**
*   **Cause:** 10,000 units/day limit reached.
*   **Fallback:**
    1. Cache all playlist metadata aggressively.
    2. Use `youtube-transcript` (no quota) for transcripts.
    3. Show warning: "YouTube quota reached. Some features limited until tomorrow."
    4. Allow manual video ID entry as workaround.

### **Scenario 4: Network Offline**
*   **Cause:** User loses internet connection.
*   **Fallback:**
    1. All cached Expeditions/Waypoints remain accessible.
    2. Disable new Expedition creation.
    3. Queue any pending AI tasks for when online.
    4. Show offline indicator in UI header.

### **Scenario 5: Invalid/Deleted Video**
*   **Cause:** Video removed from YouTube after import.
*   **Fallback:**
    1. Mark Waypoint as `is_unavailable = true`.
    2. Preserve existing Field Guide and notes.
    3. Show placeholder: "This video is no longer available."

# 🔑 API Considerations

### **YouTube Data API v3**
*   **Quota:** 10,000 units/day (free tier).
*   **Cost per operation:**
    - `playlists.list`: 1 unit
    - `playlistItems.list`: 1 unit (50 videos per call)
    - `videos.list`: 1 unit (50 videos per call)
*   **Strategy:** Batch requests, cache everything, minimize API calls.
*   **Alternative:** For basic metadata, consider scraping `oembed` endpoint (no quota).

### **Gemini API (Free Tier)**
*   **Model:** `gemini-1.5-flash` (recommended for speed/cost).
*   **Limits:** 15 requests/minute, 1 million tokens/minute.
*   **Context Window:** 1M tokens (can analyze entire playlist transcripts).
*   **Strategy:** Batch multiple video transcripts in single requests when building Atlas.

### **YouTube IFrame Player API**
*   **No Quota:** Streaming is free and unlimited.
*   **Restrictions:** Must follow YouTube ToS (no hiding branding, etc.).

# 🔐 Settings & Security

### **Storage Method**
*   **Library:** `electron-store` (encrypted JSON file in user data directory).
*   **Location:** `%APPDATA%/atlased/config.json` (Windows)
*   **Encryption:** AES-256 for sensitive values (API keys).

### **Stored Settings**
| Key | Type | Description |
|-----|------|-------------|
| `gemini_api_key` | String (encrypted) | User's Gemini API key |
| `youtube_api_key` | String (encrypted) | Optional YouTube Data API key |
| `theme` | String | `dark` (default) or `light` |
| `auto_quiz` | Boolean | Enable mid-stream quizzes (default: true) |
| `playback_speed` | Number | Default playback speed (default: 1.0) |
| `srs_enabled` | Boolean | Enable Memory Checkpoints (default: true) |
| `srs_intervals` | Array | Custom SRS intervals in days (default: [1, 3, 7, 14]) |

### **First-Run Setup**
1. App detects no API key configured.
2. Modal prompts user to enter Gemini API key.
3. Link provided to get free key: `https://aistudio.google.com/apikey`
4. Key validated with test request before saving.
